**WWDC26 Update -- Swift 6.4 Default Actor Isolation:**

Swift 6.4 introduces a new default actor isolation setting that can make `@MainActor` the default for entire modules. This is a significant quality-of-life improvement for SwiftUI-heavy codebases.

```swift
// ✅ Swift 6.4: Module-level default actor isolation
// In your Package.swift or build settings:
// defaultIsolation: @MainActor

// This makes ALL types in the module @MainActor by default
// Eliminates the need for per-type @MainActor annotations
// Particularly useful for SwiftUI view modules

// ✅ Before Swift 6.4: Explicit per-type annotation
@MainActor
class ContentViewModel: ObservableObject {
    @Published var items: [Item] = []
}

// ✅ After Swift 6.4 (with module default): No annotation needed
class ContentViewModel: ObservableObject {
    @Published var items: [Item] = []
    // Automatically @MainActor due to module default
}

// ✅ Opt out for background work
nonisolated func processData() async -> [Item] {
    // Runs off main actor, explicitly nonisolated
    return await heavyComputation()
}
```

**Swift 6.4 Ergonomics Improvements:**

Additional quality-of-life improvements in Swift 6.4:
- Better diagnostics for sendability violations (clearer error messages)
