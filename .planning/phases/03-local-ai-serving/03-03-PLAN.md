  <action>
    Set up the MLX alternative serving path:

    1. Download the MLX-native model (will auto-download on first inference, but trigger explicitly):
       ```bash
       python3 -c "from huggingface_hub import snapshot_download; snapshot_download('mlx-community/Qwen2.5-Coder-7B-Instruct-4bit', local_dir='./models/qwen2.5-coder-7b-instruct-4bit-mlx')"
       ```
       This downloads to `./models/` within the project. If the directory doesn't exist, create it.
       If `huggingface_hub` is not installed: `pip install huggingface_hub`.

    2. Verify the model loads and generates via mlx_lm CLI:
       ```bash
       cd /Users/bretbouchard/apps/me_code_pretty_one_day
       python3 -m mlx_lm.generate \
         --model ./models/qwen2.5-coder-7b-instruct-4bit-mlx \
         --prompt "def fibonacci(n):" \
         --max-tokens 100
       ```
       Confirm valid Python output.

    3. Create `scripts/serve-mlx.py` — a launch script that:
       - Starts `mlx_lm.server` as a subprocess pointing to the local model path
       - Model path: `./models/qwen2.5-coder-7b-instruct-4bit-mlx` (relative to project root)
       - Default host: 0.0.0.0, port: 8800
       - Waits for server to be ready (poll `http://localhost:8800/v1/models` until 200)
       - Prints "mlx_lm.server ready on http://localhost:8800" when healthy
       - Handles SIGINT/SIGTERM gracefully (shuts down subprocess)
       - Uses `subprocess.Popen` not `os.system` for process management
       - Accepts optional `--port` argument (default 8800)

    4. Start the server and test via API:
       ```bash
       cd /Users/bretbouchard/apps/me_code_pretty_one_day
       python3 scripts/serve-mlx.py &
       sleep 10
       curl -s http://localhost:8800/v1/chat/completions \
         -d '{
           "model": "mlx-community/Qwen2.5-Coder-7B-Instruct-4bit",
           "messages": [{"role": "user", "content": "Write a Python function to check if a string is a palindrome."}],
           "max_tokens": 100
         }' | python3 -m json.tool
       ```
       Confirm code generation output is valid Python.

    5. Stop the server after testing:
       ```bash
       pkill -f "mlx_lm.server"
       ```
