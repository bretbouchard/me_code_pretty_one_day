    Implement `scripts/prompt_template.py` with a `PromptAssembler` class. This class handles the full prompt assembly pipeline: load SLC prompt, retrieve patterns from Confucius, assemble within token budget.

    **Class: PromptAssembler**

    Constructor `__init__(self, model_path: str, max_context: int = 32768)`:
    - Load the Qwen2.5-Coder tokenizer from model_path using `transformers.AutoTokenizer.from_pretrained(model_path, trust_remote_code=True)`
    - Load SLC system prompt from `prompts/slc_system.txt` (resolve relative to PROJECT_ROOT)
    - Load domain mapping from `prompts/domains.json` (resolve relative to PROJECT_ROOT)
    - Store all as instance attributes

    **Method: retrieve_patterns(self, domain: str, limit: int = 5) -> list[dict]**
    - Look up domain in self.domains mapping. If not found, use domain string directly as query.
    - Try the first query from the domain's "queries" list via `confucius search`.
    - If no results, try the second query. If still no results, return empty list (graceful degradation).
    - For each search result, extract pattern IDs from ### headers, then call `confucius get <id>` for full JSON content.
    - Parse JSON output from `confucius get` -- extract keys: id, type, content, tags.
    - Return list of dicts: [{"id": ..., "type": ..., "content": ..., "tags": ...}, ...]
    - Use subprocess.run with timeout=10 for both search and get calls.
    - Wrap subprocess calls in try/except (subprocess.TimeoutExpired, json.JSONDecodeError, KeyError) and return empty list on failure (graceful degradation per Pitfall 4 in RESEARCH.md).

    **Method: assemble(self, user_prompt: str, domain: str = "python", max_generation_tokens: int = 2048) -> list[dict]**
    - Call self.retrieve_patterns(domain) to get RAG patterns
    - Build RAG section string: "## Retrieved Patterns\n\n" + for each pattern: "### {type.title()}\n{content}\n\n"
    - Build system_content: self.slc_prompt + "\n\n" + rag_section
    - Build messages: [{"role": "system", "content": system_content}, {"role": "user", "content": user_prompt}]
    - Apply chat template via tokenizer.apply_chat_template(messages, add_generation_prompt=True, tokenize=True)
    - Check if len(tokenized) + max_generation_tokens > max_context
    - If overflow: iteratively pop the last pattern from the list, rebuild rag_section and system_content, re-tokenize, until budget fits
    - Return the messages list (list of dicts with role/content)
